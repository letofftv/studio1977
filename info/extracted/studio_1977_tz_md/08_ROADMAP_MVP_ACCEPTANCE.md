# Этапы разработки, MVP и критерии приемки: сайт "Студия 1977"

## 1. Общий подход

Проект лучше делать поэтапно, а не пытаться сразу собрать весь идеальный продукт в один релиз.

Это позволит:
- быстрее выйти в рабочую версию
- не утонуть в бесконечном дизайне
- протестировать реальные сценарии
- нормально завести Битрикс24
- потом спокойно доращивать платформу

## 2. Этап 0. Подготовка

### Что нужно сделать
- утвердить структуру сайта
- утвердить перечень услуг
- утвердить карту кабинетов
- утвердить состав ролей
- утвердить сущности и статусы
- утвердить модель Битрикс24
- собрать референсы
- согласовать дизайн-направление
- утвердить стек

### Результат
Есть понятная проектная база, по которой можно считать сроки и стоимость.

## 3. Этап 1. Дизайн

### Что входит
- moodboard
- дизайн-концепция
- UI-kit

- дизайн кабинета клиента
- дизайн внутреннего кабинета

## 6. Что считается MVP готовым

MVP готов, если:
- сайт хорошо выглядит и не стыдно показывать
- заявки уходят в CRM
- клиент может зайти в кабинет
- клиент видит свои проекты
- может загрузить файл
- может пройти согласование
- команда видит проекты и статусы
- есть связь с Битрикс24
- сайт можно установить как web-app

## 7. Критерии приемки по блокам

## 7.1 Дизайн
- есть единая дизайн-система
- экраны отрисованы полностью
- есть мобильные версии
- есть логика состояний
- кабинеты вписаны в бренд

## 7.2 Frontend
- макеты реализованы корректно
- адаптив работает
- формы работают
- кабинеты открываются корректно
- web-app устанавливается
- базовые показатели скорости в норме

## 7.3 Backend
- роли соблюдаются
- данные сохраняются корректно
- файлы защищены
- логи пишутся
- API документирован

## 7.4 Интеграция с Битрикс24
- формы создают сущности
- связи между сущностями сохраняются
- статусы синхронизируются
- нет утечек ключей
- есть обработка ошибок

## 8. Что нужно проверить перед релизом

### UX
- понятна ли главная
- понятно ли, как оставить заявку
- удобно ли клиенту зайти в кабинет
- удобно ли согласовывать этап
- удобно ли загрузить файл с телефона

### Техника
- нет битых ссылок
- нет кривой адаптивности
- нет случайно открытых приватных страниц
- корректно работают формы
- корректно работают роли
- корректно работают уведомления
- корректно работает установка web-app

### Интеграции
- лид создается
- сделка создается или обновляется
- контакт корректно связывается
- проект на сайте привязан к CRM
- ошибки логируются

- исходники

- env-example

- описание сущностей
- инструкция по работе с Битрикс24-связкой
- описание логики кабинетов
- список известных ограничений MVP

## 10. Финальный принцип

Хороший релиз этого проекта это не просто "сайт готов".  
Хороший релиз это когда:
- студия выглядит сильно
- клиенту удобно
- команде удобно
- CRM не мешает, а помогает
- сайт работает как реальный цифровой инструмент
