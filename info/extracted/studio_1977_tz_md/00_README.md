# Пакет ТЗ для сайта проекта "Студия 1977"

## Что внутри

Этот комплект собран как рабочая база для запуска проектирования и разработки сайта студии с личными кабинетами, интеграцией с Битрикс24 и режимом web-app.

Пакет рассчитан на:
- дизайнера интерфейсов
- верстальщика
- frontend-разработчика
- backend-разработчика
- fullstack-команду
- интегратора Битрикс24
- руководителя проекта

## Логика проекта

Сайт должен решать сразу несколько задач:

1. Презентация студии как сильного визуального и продюсерского бренда.
2. Захват и прогрев заявок.
3. Показ кейсов, услуг, подхода и форматов работы.
4. Работа через личный кабинет клиента.
5. Работа через внутренний кабинет команды.
6. Синхронизация заявок, сделок, контактов, проектов и задач с Битрикс24.
7. Возможность установить сайт на телефон как web-app.

## Базовые допущения проекта

На текущем этапе принята такая модель:

- сайт студии делается как сильный имиджевый и продающий продукт
- на сайте есть кабинет клиента
- на сайте есть внутренний кабинет команды
- сайт синхронизируется с Битрикс24
- часть управления ведется в Битрикс24, но интерфейс для клиента и команды остается в самом сайте
- сайт должен быть быстрым, адаптивным и пригодным для установки на телефон
- в основе не шаблонная витрина, а кастомный цифровой продукт студии

## Что желательно утвердить следующим шагом

После изучения ТЗ желательно отдельно согласовать:
- точную карту услуг студии
- перечень типов кейсов
- структуру этапов по проектам
- что именно клиент видит в кабинете на первом релизе
- что именно команда ведет внутри сайта, а что ведет только в Битрикс24
- визуальный референс-лист с 10-20 сайтами

## Важный принцип

Этот пакет не должен восприниматься как "примерно и потом разберемся".  
Это рабочая база, которую уже можно отдавать на оценку, проектирование интерфейса и техническую декомпозицию.
