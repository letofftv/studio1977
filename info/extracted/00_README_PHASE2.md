# Студия 1977. Пакет 2. Прикладная спецификация

Этот пакет продолжает базовые ТЗ из первого архива и переводит их в рабочий формат для команды разработки.

Внутри:
- карта страниц и навигации
- роли, права и пользовательские сценарии
- карта сущностей и полей для Битрикс24
- схема БД сайта
- API-контракт
- логика кабинетов клиента и команды
- модель статусов, уведомлений и журналов действий

## Как использовать пакет

1. Дизайнер и UX смотрят `01_PAGE_MAP_AND_INFORMATION_ARCHITECTURE.md` и `02_ROLES_RIGHTS_AND_USER_FLOWS.md`
2. Frontend смотрит `01`, `02`, `05_API_CONTRACT.md`, `06_CABINETS_MODULE_LOGIC.md`, `07_STATUS_NOTIFICATIONS_AND_AUDIT.md`
3. Backend смотрит `03_BITRIX24_ENTITY_FIELD_MAP.md`, `04_DATABASE_SCHEMA.md`, `05_API_CONTRACT.md`, `07_STATUS_NOTIFICATIONS_AND_AUDIT.md`
4. Интегратор Bitrix24 в первую очередь работает с `03_BITRIX24_ENTITY_FIELD_MAP.md`
5. PM использует `02` и `07` как основу для acceptance criteria и постановки задач

## Что считается источником правды

- Публичные страницы, структура кабинетов, маршруты и пользовательские сценарии определяются этим пакетом
- Содержимое карточек, набор полей и бизнес-правила синхронизации определяются этим пакетом и первым архивом
- Если при разработке появится конфликт между дизайном и реализацией, первичен сценарий пользователя, потом данные, потом визуальная подача

## Что нужно утвердить до старта разработки

- финальный список услуг и направлений
- названия этапов проектов
- набор ролей команды
- обязательный набор полей брифа
- какие документы реально будут храниться на сайте, а какие только в Битрикс24
- будет ли клиентский чат внутри кабинета или только комментарии по проекту
- кто владелец данных и кто отвечает за актуальность статусов

## Рекомендуемая реализация

- Frontend: Next.js / React
- Backend: Node.js / NestJS или Express
- БД: PostgreSQL
- Файлы: S3-совместимое хранилище
- Auth: email + magic link / password, опционально SMS
- Интеграция: Bitrix24 REST API + входящие и исходящие вебхуки
- PWA: installable web-app, manifest, service worker, offline-fallback для части кабинета

## Результат после внедрения

Команда получает не просто красивый сайт, а рабочую систему:
- сайт продает услуги
- заявки попадают в CRM
- клиент видит проект в кабинете
- команда ведет проект в единой логике
- данные сайта и Битрикс24 синхронизированы
